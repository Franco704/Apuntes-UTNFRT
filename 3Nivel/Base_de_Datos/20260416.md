# Herramientas, Conceptos de BD y Tipos

**Adventure Works:** Una BD de Microsoft que podemos cargar, ahí podemos ver el uso de las nomenclaturas. Tenemos que mantener siempre una sola nomenclatura, no mezclar.
Una alternativa a Adventure Works es **Sakila**. De todas maneras, no hay que hacerse problemas por la nomenclatura.

> [!TIP] Recursos Útiles
> * **DB Fiddle:** Para probar secuencias SQL.
> * **Draw.io:** Para diagramas.
> * **ERDPlus:** Para diagramas.
> * **Mockaroo:** Para hacer tablas, el mocking, también para generar datos de simulación.
> * **PgAdmin:** Para gestionar la BD visualmente, para no utilizar CLI.
> * **Workbench:** Para BD de MySQL.
> * **DBeaver:** Cliente universal de base de datos, también sirve para hacer los diagramas (tiene licencia educacional). Podemos tener las tablas y rellenarlas con datos simulados (mocking).
> * **pgModeler:** (También tiene licencia educacional). Cliente para ver diagramas de PostgreSQL y puede generar datos mockeados.

## Conceptos Clave (Cosas que tenemos que saber)
* **Dato:** Hecho aislado, sin contexto ni interpretación (Ej: `10`, `"Juan"`, `20/01/2026`).
* **Información:** Dato procesado, con contexto (Ej: *Juan vendió 10 naranjas el 20/01/2026*).
* **Conocimiento:** Información interpretada con experiencia. Permite tomar decisiones (Ej: *Juan es el vendedor del día*).
* **Base de Datos (BD):** Colección organizada de datos (relacionales) almacenada de forma pertinente y que tiene que modelar el dominio del problema.
* **Motor de Base de Datos:** Software que gestiona, almacena, consulta y manipula la base de datos.
* **Tabla:** Estructura bidimensional de filas y columnas que almacena datos.
* **Registro:** Una fila completa de una tabla.
* **Campo:** Propiedad individual de un registro. Es una propiedad o atributo de una entidad.
* **Archivo de datos:** Conjunto de registros almacenados.
* **Esquema:** Conjunto de tablas.
* **Primary Key (Clave Primaria):** Campo que identifica unívocamente a un individuo de una entidad.
* **Clave Candidata:** Propiedad posibilitada a ser seleccionada como identificador único.
* **Clave Foránea:** Es la propiedad que sirve de referencia para la relación con otras entidades.

## Tipos de Bases de Datos
Existen diversos tipos de bases de datos según su estructura:
* Base de Datos Relacional (SQL)
* Documentales
* Clave-Valor
* Columnas
* Grafos
* Vectoriales
* Orientadas a objetos

> [!IMPORTANT] Enfoque Profesional
> En el ámbito profesional hay que saber principalmente de BD **Relacionales**, **Clave-Valor** y de **Columnas**.

## Notación Big O
Pasos para el análisis de complejidad:
1. Contar operaciones.
2. Quedarnos con el término dominante.
3. Eliminar constantes (Ejemplo: $5n^2 = O(n^2)$).
