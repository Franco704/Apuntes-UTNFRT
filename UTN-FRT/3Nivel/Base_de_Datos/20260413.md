El modelo entidad relación es algo ya viejo, masomenos de los años 70.
Intentaremos llevar todas las relaciones a relaciones binarias.

Las relaciones tienen cardinalidad, las cardinalidades pueden ser de uno a uno, de uno a muchos y de muchos a muchos.

La cardinalidad de uno a uno
	Una instancia de las entidades se asocia con una única instancia de otra entidad. Ej Vehículo - Empleado, a un empleado le corresponde solo un vehículo, por lo que este vehículo no puede estar asignado a ningún otro empleado.
	 Si se le pudiera asignar más de un vehículo a un empleado tendríamos uno a muchos, el caso restante sería muchos a muchos y puede tener una restricción.

Uno a N:
	A una instancia de la entidad de un tipo se relaciona a n instancias de otra entidad.

Muchos a Muchos:
	Muchas instancias de una entidad se relacionan con muchas de la otra. Esta relación es una de las que más estudiaremos para poder pasar al modelo lógico.

Hay notaciones para asignar cardinalidades, varían entre autores, tenemos que tratar de mantener una sola nomenclatura.
Las cardinalidades son en ambos sentidos, tomamos la mínima de una entidad con la máxima de otra y eso nombraremos como "la cardinalidad total".

Ahora vemos un poco de los atributos en las relaciones:
	Las entidades tienen atributos, además la relación puede tener un atributo, como por ejemplo en el caso de Dormitorio, Estudiante un atributo propio de la relación sería Alquiler, la mayoría de relaciones no tienen atributos pero puede suceder, aumenta la complejidad del modelo y veremos más adelante como trataremos dichos casos.

Entidades Débiles:
	Entidades dentro de una relación que dependen de otra entidad que llamamos "Entidad padre" o "Entidad fuerte". Por ejemplo Empleado-Departamento, el Empleado pertenece a un Departamento, la existencia de la entidad en la tabla de entidad débil depende de la existencia del de la tabla fuerte.


Entidades de tipo Rol o que se relacionan consigo mismas:
	Es un tipo especial, se relacionan consigo mismas mediante un lazo. También se llaman reflexivas. Se relacionan en la misma tabla básicamente.

Técnica para el modelado de datos utilizando diagramas entidad relación.
1. Se parte de los requisitos, una descripción textual de problema o sistema de información a automatizar.
2. Se hace una lista de sustantivos y verbos que aparecen, los sustantivos son posibles entidades o atributos, los verbos son posibles relaciones.
3. Analizando las frases se determina la cardinalidad de las relaciones y otros detalles.
4. Se elabora el diagrama (o diagramas) entidad-relación.
5. Se completa el modelo con listas de atributos y una descripción de otras restricciones que no se pueden reflejar en el diagrama.

El modelo entidad-relacion es solo el principio del modelado, podemos aplicar distintas técnicas para lograr un modelo implementable en una BD.
Alguna de ellas:
	Transformación de relaciones múltiples en binarias.
	Normalización de una base de datos de relaciones (algunas relaciones pueden transformarse en atributos y viceversa).
	Conversión en tablas (en caso de utilizar una base de datos relacional).
