## Fundamentos del Modelo

El modelo entidad-relación es algo ya viejo, más o menos de los años 70. En esta cátedra, intentaremos llevar todas las relaciones a **relaciones binarias**.

## Cardinalidades

Las relaciones tienen cardinalidad; las mismas pueden ser de **uno a uno**, de **uno a muchos** y de **muchos a muchos**. Las cardinalidades se analizan en ambos sentidos: tomamos la mínima de una entidad con la máxima de otra y eso lo nombraremos como la **"cardinalidad total"**.

### 1. Cardinalidad de Uno a Uno (1:1)

Una instancia de las entidades se asocia con una única instancia de otra entidad.

- **Ejemplo:** `Vehículo` - `Empleado`. A un `Empleado` le corresponde solo un `Vehículo`, por lo que este `Vehículo` no puede estar asignado a ningún otro `Empleado`.
    
- Si se le pudiera asignar más de un `Vehículo` a un `Empleado`, tendríamos **uno a muchos**; el caso restante sería **muchos a muchos** y puede tener una restricción.
    

### 2. Uno a Muchos (1:N)

A una instancia de la entidad de un tipo se le relaciona a n instancias de otra entidad.

### 3. Muchos a Muchos (N:N)

Muchas instancias de una entidad se relacionan con muchas de la otra. Esta relación es una de las que más estudiaremos para poder pasar al modelo lógico.

---

## Atributos en las Relaciones

Las entidades tienen atributos; además, la relación puede tener un atributo propio.

- **Ejemplo:** En el caso de `Dormitorio` y `Estudiante`, un atributo propio de la relación sería `alquiler`.
    
- La mayoría de las relaciones no tienen atributos, pero puede suceder. Esto aumenta la complejidad del modelo y veremos más adelante cómo trataremos dichos casos.
    

---

## Entidades Especiales

### Entidades Débiles

Son entidades dentro de una relación que dependen de otra entidad que llamamos **"Entidad padre"** o **"Entidad fuerte"**.

- **Ejemplo:** `Empleado` - `Departamento`. El `Empleado` pertenece a un `Departamento`; la existencia de la entidad en la tabla de **entidad débil** depende de la existencia del registro en la **tabla fuerte**.
    

### Entidades de tipo Rol (Reflexivas)

Es un tipo especial; las entidades se relacionan consigo mismas mediante un lazo. Se relacionan en la misma tabla básicamente.

## Técnica para el Modelado de Datos

Para utilizar diagramas entidad-relación, seguiremos estos pasos:

1. **Requisitos:** Partir de una descripción textual del problema o sistema de información a automatizar.
    
2. **Identificación:** Hacer una lista de sustantivos (posibles entidades o atributos) y verbos (posibles relaciones).
    
3. **Análisis:** Determinar la cardinalidad de las relaciones y otros detalles analizando las frases.
    
4. **Diagramado:** Elaborar el diagrama (o diagramas) entidad-relación.
    
5. **Complemento:** Completar el modelo con listas de atributos y una descripción de otras restricciones que no se pueden reflejar en el diagrama.
    

> [!INFO] Evolución del Modelo El modelo entidad-relación es solo el principio; podemos aplicar distintas técnicas para lograr un modelo implementable en una BD:
> 
> - Transformación de relaciones múltiples en binarias.
>     
> - Normalización de una base de datos de relaciones (algunas relaciones pueden transformarse en atributos y viceversa).
>     
> - Conversión en tablas (en caso de utilizar una base de datos relacional).
