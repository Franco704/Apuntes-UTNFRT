## 📚 Conceptos Fundamentales
Las bases de datos vienen a ayudarnos al tratamiento, almacenamiento y consultas.
Los datos procesados nos dan **información**. Para que los datos puedan ser procesados debemos organizar lógicamente los archivos.

> [!QUOTE] Base de Datos
> Colección de archivos interrelacionados creados y administrados por un DBMS.

> [!QUOTE] Sistemas de Bases de Datos
> Es un sistema computarizado de información para el manejo de datos por medio de paquetes de software llamado Sistemas de Administración de Base de Datos. Sus componentes son: Hardware, Software y Datos.

**DBMS (Data Base Management System):**
Es un sistema que tiene un gestor de datos, donde podemos modificar los datos; su objetivo es facilitarnos la gestión de datos. Los motores de DBMS (las marcas comerciales) tienen diversas funciones:
- Crear y organizar bases de datos.
- Manejo de transacciones y control de concurrencia o de los accesos.
- Manejar los datos de acuerdo a la petición de los usuarios.
- Control de acceso.

**Transacción:** Significa una petición en línea de la base de datos, implica llamadas a rutinas del DBMS para operaciones de E/S y alguna cantidad limitada de operaciones.

## 🎯 Objetivos de un sistema de base de datos
1. **Independencia de Datos:** Física y Lógica.
2. **Minimizar la Redundancia de datos.**
3. **Gasto de capacidad de almacenamiento.**
4. **Integridad de los Datos:** Medidas de seguridad usadas para mantener correctos los datos en la base de datos.
5. **Control de concurrencia y simultaneidad.**
6. **Seguridad de los Datos.**

## ⚖️ Habeas Data (Lectura Asignada)
Es un **derecho fundamental y una acción judicial** que permite a cualquier persona conocer, actualizar, rectificar o eliminar los datos personales que consten en archivos, bancos de datos o registros públicos o privados. 
- Protege el honor, la intimidad y la autodeterminación informativa. 
- Se utiliza cuando se incurre en falsedad, discriminación o tratamiento inadecuado de datos, o cuando la información es obsoleta (derecho al olvido). 
- En Argentina, este derecho tiene rango constitucional (artículo 43) y está regulado por la **Ley 25.326 de Protección de Datos Personales**.

## 🏗️ Arquitectura Funcional
Una arquitectura funcional estándar se divide en tres grandes capas:
### 1. Procesador de Consultas (Query Processor)
Es la capa superior, la que interactúa directamente con los usuarios o las aplicaciones externas. Su trabajo es traducir las consultas en lenguaje de alto nivel (como SQL) a instrucciones de bajo nivel que la máquina pueda entender.
- **Analizador (parser/compiler):** Verifica que la consulta tenga la sintaxis correcta y que las tablas o columnas solicitadas realmente existan consultando el *Diccionario de Datos*.
- **Optimizador de consultas:** Es el cerebro de la capa. Evalúa múltiples formas de ejecutar la consulta y elige el plan de ejecución más eficiente para ahorrar tiempo y recursos.
- **Motor de ejecución:** Toma el plan elegido por el optimizador y lo ejecuta, haciendo las llamadas necesarias al Gestor de Almacenamiento para recuperar los datos.

### 2. Gestor de Almacenamiento (Storage Manager)
En esta capa la lógica se encuentra con el hardware, es la responsable de la interacción con los datos guardados en el disco y de minimizar los accesos al almacenamiento secundario transfiriendo información hacia la memoria principal (RAM).
- **Gestor de Búfer (Buffer Manager):** Decide qué datos deben mantenerse en la memoria caché para un acceso rápido y cuáles deben escribirse de vuelta al disco. Es vital para el rendimiento.
- **Gestor de Archivos:** Administra el espacio en disco, llevando el control de dónde se ubican exactamente los bloques o páginas (registros físicos) y gestionando el espacio libre.
- **Estructuras de Datos y Métodos de Acceso:** Interactúa con los índices y las tablas Hash para localizar rápidamente los registros lógicos solicitados sin tener que leer archivos desordenados.

### 3. Gestor de Transacciones (Transaction Manager)
Esta capa garantiza que la base de datos sea confiable y segura, asegurando que se cumplan las propiedades **ACID** (Atomicidad, Consistencia, Aislamiento, Durabilidad).
- **Control de Concurrencia:** Se asegura de que si múltiples usuarios intentan modificar los mismos datos al mismo tiempo, las operaciones se ordenen correctamente para evitar inconsistencias.
- **Gestor de Recuperación:** Mantiene un archivo de registro (*Log*) con todas las operaciones. Si el servidor se apaga repentinamente a mitad de una actualización, este componente usa el log para deshacer los cambios incompletos o rehacer los confirmados cuando el sistema vuelva a encenderse.

**Elementos físicos clave en el disco:**
1. **Archivo de Datos:** Donde viven los registros reales.
2. **Archivos de Índices:** Las estructuras que aceleran la búsqueda.
3. **Diccionario de Datos (Metadatos):** Datos sobre los datos (estructura de tablas, permisos, esquemas).

## 📊 Modelo de Datos
> [!QUOTE] Definición
> Es un grupo de herramientas conceptuales para describir los datos, sus relaciones, su semántica y sus limitaciones; de tal forma que facilita la interpretación de nuestro mundo real y su representación en forma de datos, en nuestro sistema informático.

**Tipos:**
* **Basados en Objetos:**
	* Entidad Relación (*)
	* Semántico
	* Infologico
	* Funcional de Datos
	* Binario
	* ODL
* **Basados en Registros:**
	* Relacional (*)
	* Jerárquico
	* Red
