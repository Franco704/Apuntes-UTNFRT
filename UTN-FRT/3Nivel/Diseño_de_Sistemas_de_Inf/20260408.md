#  Introducción al Diseño de Sistemas de Información

Es imposible operar el mundo moderno sin software; las infraestructuras de servicios públicos se controlan mediante sistemas basados en computadoras, muchos productos eléctricos incluyen computadora y software de control, la fabricación y distribución industrial están completamente computarizadas, y el entretenimiento usa software de manera intensiva. La ingeniería de software es esencial para el funcionamiento de las sociedades.

Los sistemas de software son abstractos e intangibles, no están regidos por leyes físicas ni por procesos de fabricación. Sin embargo, pueden volverse rápidamente muy complejos, difíciles de entender y costosos de cambiar.

### Causas de Fallas en el Software
Muchas fallas del software son consecuencia de dos factores principales:
1. **Demandas Crecientes:** Los sistemas deben construirse y distribuirse más rápidamente; se requieren sistemas más grandes y más complejos con nuevas capacidades.
2. **Expectativas Bajas:** Es sencillo programar sin usar métodos y técnicas de ingeniería de software. 
	* Muchas compañías recién incurren en la ingeniería de software cuando evolucionan sus productos y servicios.
	* Al no usar ingeniería de software, su software suele ser más costoso y menos confiable. Es necesaria una mejor educación y capacitación para solucionar este problema.

> [!NOTE] Desarrollo de Software Profesional
> El desarrollo de software es una actividad profesional, el software se realiza para propósitos de negocios específicos, su inclusión en otros dispositivos o como productos de software. 
> El software profesional (destinado a que lo use alguien más) **se lleva a cabo por equipos** y **se mantiene y cambia a lo largo de su vida**.

Un sistema de software desarrollado profesionalmente es más que un programa, consta de programas separados y archivos de configuración. Incluye documentación del sistema, documentación del usuario y sitios web de soporte.

## Diferencias Clave

### Diseño de Sistemas vs Diseño de Software (Aficionado vs Profesional)
* **Aficionado:** Si diseñamos un programa personal, nadie más lo usará, entonces no hay que elaborar guías del programa ni documentar el diseño.
* **Profesional:** Si creamos un software que otros usarán y otros ingenieros cambiarán, se debe ofrecer información adicional así como el código de programa. 

### Tipos de productos de Software
1. **Productos genéricos:** Sistemas independientes que se producen y venden en el mercado a cualquier cliente. La organización que desarrolla el software controla su especificación.
2. **Productos personalizados (a medida):** Sistemas destinados para un cliente en particular, desarrollados por un contratista. La organización que compra el SW desarrolla y controla la especificación.

### Ingeniería del Software vs Otras Disciplinas
1. **Ciencias de la Computación:** Estudia teorías y métodos que subyacen en computadoras y sistemas de software. (Las teorías no siempre se aplican a grandes problemas complejos).
	* *La Ing. de Software* vela por los asuntos prácticos de la producción.
2. **Ingeniería de Sistemas:** Estudia los aspectos del desarrollo y evolución de complejos sistemas (hardware, políticas, procesos) donde el software tiene un papel principal.
	* Intervienen en la especificación del sistema, definen su arquitectura global y luego integran las partes.

## Ingeniería del Software: Conceptos Fundamentales

> [!QUOTE] Definición
> Disciplina de la ingeniería que se interesa por todos los aspectos de la producción de software, desde las primeras etapas de la especificación hasta el mantenimiento del sistema.

**Frases clave de la definición:**
1. **Disciplina de ingeniería:** Los ingenieros hacen que las cosas funcionen aplicando métodos de manera selectiva, trabajando con restricciones organizacionales y financieras.
2. **Todos los aspectos de la producción:** Se interesa por procesos técnicos, administración de proyectos y desarrollo de herramientas.

### Las 4 actividades comunes a los Procesos de Software
1. **Especificación del software (Análisis):** Define el software que se producirá y las restricciones en su operación.
2. **Desarrollo del software:** Se diseña y programa el software.
3. **Validación del Software (Prueba):** Se verifica para asegurar que sea lo que el cliente requiere.
4. **Evolución del software (Mantenimiento):** Se modifica para reflejar los requerimientos cambiantes del cliente y del mercado.

### 3 Problemas principales que afectan al software actual
1. **Heterogeneidad:** Se requieren sistemas que operen en redes con diferentes computadoras y móviles, integrando software nuevo con sistemas viejos.
2. **Cambio empresarial y social:** Los negocios cambian rápidamente; hay que reducir el tiempo de entrega para que el software dé valor rápido a los clientes.
3. **Seguridad y confianza:** Es esencial confiar en el software, sobre todo en sistemas web, evitando ataques de malintencionados.

## Estrategias de Prototipado

* **Prototipo:** Versión inicial de un sistema de software.
* **Prototipo de sistema:** Estrategia para desarrollar rápidamente una versión del sistema para comprobar requerimientos y factibilidad de diseño.

**¿Para qué se usa?**
* Permite que los usuarios experimenten con el sistema antes de entregarlo para refinar sus requerimientos.
* En el **Análisis**, ayuda con la selección y validación de requerimientos.
* En el **Diseño**, busca soluciones y apoya el diseño de interfaces de usuario.

![Modelo de Proceso de Prototipos](../../Bibliografia/Pasted%20image%2020260408185053.png)

### Evaluación y Riesgos del Prototipo
* Hay que capacitar al usuario y usar los objetivos del prototipo para evaluarlo, ya que requieren tiempo para sentirse cómodos.
* **El peligro de los prototipos desechables:** A veces los desarrolladores están presionados para entregar el prototipo como sistema final, lo cual es un error porque:
	1. Es difícil corregir requerimientos no funcionales ignorados.
	2. No está documentado (difícil mantenimiento).
	3. Los cambios rápidos degradan la estructura del sistema.
	4. Se relajan los estándares de calidad.

##  Fundamentos aplicables a todo tipo de sistemas
1. **Usar un proceso de desarrollo administrado y comprendido:** Planear el proceso y tener ideas claras sobre lo que se producirá.
2. **Priorizar confiabilidad y desempeño:** El software debe comportarse como se espera y estar disponible.
3. **Comprender y gestionar la especificación:** Saber exactamente qué debe hacer el software.
4. **Usar de forma efectiva los recursos.**
